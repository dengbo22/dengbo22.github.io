Android列表控件RecyclerView

发表于 2015-12-16   |  

RecyclerView中的相关类：
RecyclerView.Adapter: 可以托管数据集合，为每一项Item创建视图并且绑定数据
RecyclerView.ViewHolder: 承载Item视图的子布局
RecyclerView.LayoutManager: 负责Item视图的布局的显示管理
RecyclerView.ItemDecoration: 给每一项Item视图添加子View,例如可以进行画分隔线之类的
RecyclerView.ItemAnimator: 负责处理数据添加或者删除时候的动画效果

1. 使用RecyclerView

compile ‘com.android.support:recyclerview-v7:23.1.1’

2.重载Adapter

对于Adapter而言，有几个关键函数需要重载

  • public int getItemCount()
    个人理解来看，这个函数将会被首先调用，从而RecyclerView将会根据这个函数判断需要创建多少个ViewItem，一般都是返回Adapter对应内容的List的size()

  • public int getItemViewType(int position)
    接下来将根据获取的item数量逐个创建item。这个函数用来获取ListView在指定position的Type，从而系统或根据获取到的Type类型来决定是重用已经用过的ViewHolder还是重新新建一个ViewHolder。

  • public RecyclerView.ViewHolder onCreateViewHolder(ViewGroup parent, int typeView)
    个人理解：此处parent就是之前在RecyclerView中创建的一个一个的item，而后根据前一个函数获取到的ViewType，从而加载不同的内容，并且将形成的整个视图作为ViewHolder返回出去。所以此处主要执行inflate函数，并且将结果view作为传给各个ViewHolder作为参数执行findViewById()，并将内容保存在ViewHolder的内部类里面。

  • public void onBindViewHolder( RecyclerView.ViewHolder viewHolder, int position)
    该个函数将会根据上一个函数所创建的ViewHolder，并且将这个ViewHolder作为参数传递到这个函数中，所以可以根据参数中的position得到类型名称并且将获取到的ViewHolder的类型进行强制类型转换，并且将具体的View的内容进行设置，相当于在Fragment中调用getTag()函数以后进行的操作。

  • 其他注意事项
    创建RecyclerView中的各个子类的ViewHolder，均继承自RecyclerView.ViewHolder。各个类均只包括其中的各个控件，并且重写构造函数，参数为View（其实就是parentView）。并且在构造的时候使用findViewById()将各个xml控件和java控件绑定。

3. 关于LayoutManager

布局管理器，其用来管理每一项View在RecyclerView中的位置布局以及其显示和隐藏。RecyclerView默认提供了三种内部布局管理器：

  • LinearLayoutManager： 线性布局
    其可以使用setOrientation来设置RecyclerView的水平或者竖直摆放

  • GridLayoutManager: 表格布局

  • StaggeredGridLayoutManager: 瀑布流式布局
